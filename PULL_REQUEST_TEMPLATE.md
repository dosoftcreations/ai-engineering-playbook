# Contributing

Thanks for considering a contribution.

This repository is not a prompt dump. Contributions should improve repeatable engineering practice for AI-assisted software development.

## Good contributions

Good contributions include:

- Clear engineering standards
- Practical playbooks
- Verification workflows
- Production checklists
- Model-routing guidance
- Token/context management strategies
- Agent anti-patterns
- Case studies with measurable results

## Poor contributions

Avoid:

- Generic prompts with no engineering context
- Model hype
- Tool-specific marketing
- Large vague documents
- Advice that cannot be applied
- Rules that cannot be verified

## Contribution format

When proposing a new rule, include:

1. Problem
2. Rule
3. Why it matters
4. Trade-offs
5. Good example
6. Bad example
7. Verification method if applicable

## Tone

Be direct, practical, and evidence-seeking.

Prefer:

```text
Use one subagent for medium tasks only when the scope is independent.
```

Avoid:

```text
Always use agents to maximize productivity.
```

## Versioning

This project uses semantic versioning once stable.

During alpha, breaking structural changes are allowed.
