# Anti-Patterns

This document captures common habits that reduce the quality of AI-assisted engineering work.

## Common anti-patterns

- treating AI as a replacement for judgment
- using broad prompts when a narrow task would work better
- shipping changes without relevant verification
- combining unrelated changes in one submission
- relying on hype rather than evidence

## How to avoid them

- keep work scoped and explicit
- verify important outcomes before release
- separate implementation from review when needed
- prefer small, reviewable changes
