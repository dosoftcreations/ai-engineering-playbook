# Security Policy

This repository contains engineering guidance, not production software.

However, security-related contributions should be handled carefully.

## Reporting security issues

If you find guidance that could create unsafe production behavior, open an issue with:

- Affected file
- Risk description
- Example failure case
- Safer recommendation

## Security standard

See:

```text
standards/SECURITY_STANDARD.md
checklists/SECURITY.md
```

## Principle

AI-generated software must be held to at least the same security standard as human-written software.

AI does not reduce accountability.
