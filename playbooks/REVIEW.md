# Review Playbook

Use this playbook when reviewing a change for implementation quality.

## Review focus

- correctness
- security
- maintainability
- observability
- rollback readiness

## Process

1. Review the change against the stated intent.
2. Look for hidden risk, missing validation, or broad scope.
3. Summarize the result with clear findings and recommendations.
4. Decide whether the change is safe to merge.
