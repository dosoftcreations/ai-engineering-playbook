# Code Review Playbook

Use this playbook when reviewing a change for correctness, safety, and maintainability.

## Review objectives

- Confirm the change matches the stated scope.
- Identify correctness and production-risk issues.
- Check whether the change is secure and rollback-ready.
- Ensure the change is understandable to future maintainers.

## Review workflow

1. Read the summary and the acceptance criteria.
2. Review the changed files with the relevant context.
3. Look for security, reliability, and maintainability issues.
4. Provide findings in a clear and actionable format.
5. Decide whether the change is ready for merge, needs follow-up, or should be blocked.
