# New Project Playbook

Use this playbook when starting a new software project with AI-assisted engineering.

## Suggested workflow

1. Define the product objective and the initial constraints.
2. Choose the architecture and delivery approach that fit the scope.
3. Set a minimal initial plan with clear milestones.
4. Establish baseline engineering practices for review, security, and verification.
5. Start with a narrow first implementation slice instead of a broad rewrite.

## Maintainer guidance

- Favor simple architecture over premature complexity.
- Keep the first release small and measurable.
- Document assumptions and major decisions early.
- Build review, rollback, and monitoring practices into the first iteration.
