# Planning Playbook

Use this playbook when turning an ambiguous request into an implementation plan.

## Workflow

1. Clarify the objective and the acceptance criteria.
2. Identify the constraints, risks, and dependencies.
3. Break the work into a small set of concrete steps.
4. Define the verification strategy before implementation begins.
5. Capture assumptions and known unknowns explicitly.

## Output

A useful plan should be specific enough to guide implementation and review without overcommitting to uncertain details.
