# Pull Request Template

## Summary

Describe the change and the problem it solves in one concise paragraph.

## Scope

- What changed
- What did not change
- Why this scope is appropriate
- Any related files or follow-up work

## Verification

List the checks, tests, or review steps completed. Include evidence when available.

## Risks

Document any known risks, limitations, or rollout concerns.

## Checklist

- [ ] I reviewed [checklists/PRE_PR.md](../checklists/PRE_PR.md)
- [ ] I used a scoped, reviewable change set
- [ ] I documented any security or rollout concerns
- [ ] I linked related documentation where useful
