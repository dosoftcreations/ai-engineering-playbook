# Agent Instructions

Use this repository as a practical handbook for AI-assisted engineering work.

## Operating principles

- Prefer small, reviewable changes over broad rewrites.
- Keep the scope clear and bounded.
- Verify claims and changes when evidence is available.
- Treat security, review quality, and rollback as first-class concerns.
- Avoid adding hype, speculation, or unrelated refactors.

## Repository guidance

- Start with the standards in [standards/](standards/).
- Use the playbooks in [playbooks/](playbooks/) for recurring workflows.
- Apply the checklists in [checklists/](checklists/) before commit, PR, release, or production rollout.
- Reuse the templates in [templates/](templates/) when the workflow is consistent.

## Working style

- Ask for clarification when the task is ambiguous.
- Keep context focused on the current problem.
- Summarize progress and risks clearly.
- Document assumptions and verification evidence.
