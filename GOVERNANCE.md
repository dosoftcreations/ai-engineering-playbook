# Governance

This repository is maintained as an open-source engineering reference for practical AI-assisted software development.

## Maintainer role

Maintainers are responsible for:

- keeping the repository focused on practical engineering guidance
- preserving a consistent structure across documents
- reviewing contributions for clarity, correctness, and usefulness
- resolving disputes based on repository principles rather than personal preference

## Decision process

Major changes should be discussed in issues before implementation when they affect:

- repository structure
- core standards
- release and governance policy
- the intended scope of the project

## Scope boundaries

This repository should remain focused on:

- engineering practices for AI-assisted software development
- production-oriented guidance
- reviewable and reusable documentation

It should not become a general AI hype repository or a loose collection of prompts.
