# Example: Next.js Workflow

This example shows how the repository guidance can apply to a Next.js project.

## Example workflow

1. Start with a narrow feature or fix.
2. Keep the implementation scoped to the relevant route or component.
3. Verify the behavior with the relevant test or smoke check.
4. Review the change for security and rollout impact.

## Lessons

- Small increments are easier to verify.
- Production readiness should be considered early.
- Documentation and review should accompany the implementation.
