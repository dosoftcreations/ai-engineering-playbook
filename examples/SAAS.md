# Example: SaaS Workflow

This example shows how the repository guidance can apply to a SaaS product.

## Example workflow

1. Define the user-facing change and the rollout expectation.
2. Review permission, billing, and observability implications.
3. Verify the change with the relevant smoke tests and monitoring.
4. Prepare a rollback or containment plan if the release introduces risk.

## Lessons

- SaaS changes should be reviewed as operational changes, not only as feature changes.
- Reliability and rollback readiness are part of delivery.
- Security and billing review should be explicit for customer-facing features.
