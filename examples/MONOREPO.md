# Example: Monorepo Workflow

This example shows how the repository guidance can apply to a monorepo.

## Example workflow

1. Identify the affected package or service.
2. Keep the change scoped to the relevant boundaries.
3. Review integration points before merging.
4. Verify the change with the relevant package-level and cross-package checks.

## Lessons

- Clear ownership reduces review friction.
- Cross-package impact should be reviewed explicitly.
- Rollout plans matter even for internal refactors.
