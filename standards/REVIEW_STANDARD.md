# Review Standard

Reviews should focus on production risk, correctness, and maintainability.

## Review objectives

- Identify correctness issues.
- Identify security and reliability risks.
- Check whether the change matches the stated scope.
- Confirm that verification is possible.

## Review format

Each finding should include:

- severity
- file or area
- issue
- why it matters
- how to verify
- recommended fix

## Review verdicts

Use one of the following outcomes:

- PASS
- PASS WITH NOTES
- BLOCKED

Do not block on style preferences when the change is otherwise sound.
