# Tool Usage Standard

AI tools should be used deliberately, with clear intent and appropriate boundaries.

## Core expectations

- Use the tool that fits the task, not the tool that is most impressive.
- Keep prompts scoped to the actual objective.
- Avoid tool chaining when a single direct action would be clearer.
- Preserve human review for anything that affects production risk.

## Practical guidance

- Use automation for repetitive or mechanical tasks.
- Use reasoning support for planning, review, and trade-off analysis.
- Use code modification tools only when the requested change is well understood.
- Stop and reframe the request when the tool output becomes too broad or too ambiguous.
