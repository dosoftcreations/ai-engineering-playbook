# Reasoning Policy

This document defines how reasoning work should be approached when using AI assistance.

## Principles

- Start from a clearly stated problem.
- Separate facts from assumptions.
- Prefer explicit reasoning over hidden intuition.
- Surface uncertainty instead of pretending confidence.
- Avoid broad speculation when the evidence is incomplete.

## Recommended workflow

1. Define the problem and the constraints.
2. Identify the available evidence.
3. Propose a small set of viable options.
4. Compare the trade-offs explicitly.
5. Choose the safer and simpler path when uncertainty remains.

## When to escalate

Escalate to a human reviewer when the decision affects:

- security or authorization
- data integrity or backups
- customer-facing behavior
- large-scale architecture changes
- rollout or incident response
