# Standards

> Stable engineering rules, policies, and budgets governing AI-assisted software development in this repository.

---

## 1. Situational Guide

Use this table to find the appropriate standard document for your current task or situation:

| If you are... | Use this Document | Why it matters |
|---|---|---|
| **Starting any new coding task** | [AI_ENGINEERING_STANDARD.md](AI_ENGINEERING_STANDARD.md) | Defines core lifecycle phases, editing rules, and stop rules. |
| **Selecting which model or agent to run** | [MODEL_SELECTION.md](MODEL_SELECTION.md) | Maps task types to capability tiers to minimize cost. |
| **Budgeting token costs or file reads** | [TOKEN_ECONOMY.md](TOKEN_ECONOMY.md) | Sets hard ceilings on API consumption. |
| **Managing active memory or context size** | [CONTEXT_MANAGEMENT.md](CONTEXT_MANAGEMENT.md) | Avoids context pollution and model reasoning degradation. |
| **Spawning or coordinating subagents** | [SUBAGENT_STANDARD.md](SUBAGENT_STANDARD.md) | Outlines delegation limits and subagent output templates. |
| **Reviewing a pull request or code change** | [REVIEW_STANDARD.md](REVIEW_STANDARD.md) | Defines severity levels and generic review checklists. |
| **Handling API keys, auth, or input validation** | [SECURITY_STANDARD.md](SECURITY_STANDARD.md) | Establishes the project's security baseline. |
| **Formatting commits, branches, or git trees** | [GIT_STANDARD.md](GIT_STANDARD.md) | Keeps git trees clean, traceable, and revertible. |
| **Fleshing out logic or architectural choices** | [REASONING_POLICY.md](REASONING_POLICY.md) | Governs structured thinking and when to escalate to humans. |
| **Executing commands or running web lookups** | [TOOL_USAGE_STANDARD.md](TOOL_USAGE_STANDARD.md) | Limits tool usage to safe, non-destructive actions. |
| **Connecting external APIs or local MCP servers** | [MCP_STANDARD.md](MCP_STANDARD.md) | Governs Model Context Protocol integration safety. |
