# Git Standard

Use Git in a way that preserves clarity, reviewability, and traceability.

## Expectations

- Keep commits focused and meaningful.
- Avoid mixing unrelated changes in a single commit.
- Use descriptive commit messages and pull request summaries.
- Prefer small, reviewable diffs over large bundled changes.

## Review guidance

Before merging, confirm that the change is:

- scoped correctly
- easy to understand
- safe to review
- easy to roll back if needed
